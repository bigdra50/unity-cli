# Unity Skills ルーティングルール

## 優先ルール

Unity プロジェクトで作業する際は、以下のルーティングを自動的に適用する。

### Hook トリガー時

unity-skill-eval-hook が発動した場合:

1. 質問タイプを分析
2. 対応するスキルを Skill() ツールで読み込み
3. スキルの指示に従って作業を実行

### スキル優先順位

同時に複数のシグナルがある場合:

1. エラー調査 (/udebug) - 最優先
2. 検証 (/uverify) - テスト・コンパイル
3. その他のワークフロースキル

### エージェント使用条件

- API 詳細・ドキュメント参照 → unity-guide エージェント (Task ツール)
- バージョン情報 → version-checker エージェント (Task ツール)

## unity-cli 使用ルール

### 前提条件

- Unity プロジェクトで作業中であること
- Relay サーバー (unity-relay) が起動していること
- Unity エディタが接続されていること

### 接続確認

作業開始時に `u instances` で接続状態を確認する。

### YAML フォールバック

unity-cli で対応できない操作は YAML 直接編集:

- .meta, .asset, .prefab, .unity ファイル
- unity-guide エージェントで形式を確認してから編集

## 禁止事項

- コーディング規約の押し付け（ユーザーの rules を優先）
- 過度な抽象化の提案（ユーザーが求めない限り）
- 確認なしの破壊的操作
